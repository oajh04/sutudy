# 디렉티브

디렉티브는 `v-`로 시작하는 특수한 속성입니다. 디렉티브 속성 값은 단일 js 표현식이 됩니다. (v-for과 v-on은 예외입니다) 디렉티브의 역할은 표현식의 값이 변경될 떄 발생하는 부수 효과들을 반응적으로 DOM에 적용하는 것입니다. 

```html
<p v-if="isShow">Hello World!</p>
```

여기서 `v-if`디렉티브는 표현식 isShow 값이 참 또는 거짓 여부를 바탕으로 `<p>` 엘리먼트를 삽입하거나 제거합니다.



## 전달인자

일부 디렉티브는 디렉비트명 뒤에 콜론으로 표기되는 전달인자를 가질 수 있습니다. 예를 들어, `v-bind`디렉티브는 반응적으로 HTML 속성을 갱신하는데 사용합니다.

```html
<a v-bind:href="url">. . .</a>
```

여기서 href는 전달인자로, v-bind 디렉티브가 표현식 url의 값을 엘리먼트 href속성에 바인딩하도록 지시합니다.

```html
<a v-on:click="clickEvent"> . . . </a>
```

이 예제는 DOM 이벤트를 수신하는 `v-on`디렉티브입니다. 여기서 수신할 이벤트명이 전달인자입니다.



## 동적 전달인자

JS표현식을 대괄호로 묶어 디렉티브 전달인자로 사용할 수도 있습니다.

```html
<a v-bind:[attributeName]="url"> ... </a>
```

여기서 `attributeName`은 js 표현식으로 동적 변환되며, 변환된 결과는 전달인자의 최종값으로 사용됩니다.

```html
<a v-on:[eventName]="evnet"> ... </a>
```

위 예제에서 `eventName`의 값이 `"focus"` 라면, `v-on:[eventName]`은 `v-on:focus`와 같습니다.



### 값의 제약

동적 전달인자는 null을 제외하고 string으로 변환되어야 합니다. 특수 값 null을 사용하면 명시적으로 바인딩을 제거할 수 있습니다. 그 외의 값은 경고를 출력합니다

```html
<a v-bind:['foo' + bar]="value"> ... </a>
```

in-DOM 템플릿(HTML 파일 내에 템플릿을 직접 작성한 경우)을 쓰면 브라우저가 강제로 속성명을 소문자로 만들어 버리므로 속성명에 대문자를 넣지 않도록 합니다.



### 형식의 제약

동적 전달인자 형식에는 일부 문자상의 제약이 있습니다. HTML 속성명에 공백이나 따옴표와 같은 특정 문자가 유효하지 않기 때문입니다. 다음 예시는 잘못된 경우입니다.





## 수식어

수직어는 점으로 표기되는 특수 접미사로, 디렉티브를 특별한 방법으로 바인딩해야함을 나타냅니다. 예를 들어, `.prevent`수식어는 `v-on`디렉티브가 트리거된 이벤트에서 `event.preventDefault()`를 호출하도록 지시합니다.

```html
<form v-on:submit.prevent="onSubmit">...</form>
```
